# 启发式7：先结论后逻辑

## 一句话总结
分析报告 = **买/卖/观望 + 原因一、二、三**。

## 核心逻辑
```
先把结论说清楚
再给逻辑支撑
—— 受众不需要猜测你的观点
```

## 表达模板

```
结论：买入/卖出/观望

原因一：...
原因二：...
原因三：...
```

## 经典语录

> 「我做分析报告从来不绕弯子。先告诉你是买还是卖，然后告诉你为什么。逻辑可以很复杂，但结论必须简单。」

> 「很多人分析了一大堆，最后结论是'可能涨也可能跌'。这种报告没有意义。」

## 正确示范

```
结论：买入

原因一：ETF连续净申购3天，显示机构态度转多
原因二：股价回踩20日均线获得支撑，弹簧信号出现
原因三：赔率3:1，向下空间有限，向上空间充足
```

## 错误示范

```
"这个股票可能会有一定的上涨空间，但也要注意风险，
因为市场可能会有波动..." （没有明确结论）
```

## 决策类型

| 结论 | 使用场景 |
|------|---------|
| 买入 | 明确的买入机会，赔率足够 |
| 卖出 | 明确的卖出信号，或止损必须执行 |
| 观望 | 逻辑不清晰，或赔率一般 |

## 关键原则

- **结论必须明确**：不能模棱两可
- **逻辑不超过3条**：多了说明你自己也不确定
- **先说结论**：让受众第一时间知道你的立场
